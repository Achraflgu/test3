   🔧 ADD DATABASE_URL TO RAILWAY - DETAILED STEPS
═══════════════════════════════════════════════════════════════════════════

STEP 1: OPEN RAILWAY DASHBOARD
-------------------------------
1. Go to: https://railway.app/dashboard
2. You should see your project "test3" or "Achraflgu/test3"
3. Click on it


STEP 2: FIND THE SERVICE
-------------------------
1. You'll see a card/box with your deployment
2. It might say "Web Service" or show your GitHub repo name
3. Click on that service card


STEP 3: CLICK "VARIABLES" TAB
------------------------------
Look at the top tabs:
• Deployments
• Settings
• Metrics
• VARIABLES <-- CLICK THIS ONE
• Logs


STEP 4: ADD NEW VARIABLE
-------------------------
1. You should see "Environment Variables" or "Variables" section
2. Click the button that says:
   - "New Variable" OR
   - "Add Variable" OR
   - "+ Variable"


STEP 5: ENTER THE VARIABLE
---------------------------
Two fields will appear:

Field 1 - Variable Name:
DATABASE_URL

Field 2 - Variable Value (COPY THIS ENTIRE LINE):
postgresql://neondb_owner:npg_3qhtrFepc5wj@ep-green-forest-ag0gs7i6-pooler.c-2.eu-central-1.aws.neon.tech/neondb?sslmode=require


STEP 6: SAVE THE VARIABLE
--------------------------
1. Click "Add" or "Save" button
2. You should see the variable appear in the list
3. It will show:
   DATABASE_URL = postgresql://neondb_owner:npg_3qhtrFepc5wj@...


STEP 7: TRIGGER REDEPLOY
-------------------------
Option A (Automatic):
  - Railway should automatically redeploy when you add a variable
  - You'll see "Deploying..." at the top

Option B (Manual - if it doesn't auto-deploy):
  1. Go to "Deployments" tab
  2. Click the "..." menu on the latest deployment
  3. Click "Redeploy"

OR
  1. Go to "Settings" tab
  2. Scroll down
  3. Click "Redeploy" button


STEP 8: WAIT FOR DEPLOYMENT
----------------------------
1. Watch the deployment logs
2. Wait 2-3 minutes
3. Look for "Build successful" or "Deployed"


STEP 9: CHECK YOUR SITE
------------------------
1. Go to your Railway domain URL
2. Refresh the page (Ctrl+F5 or Cmd+Shift+R)
3. The error should be gone! ✅

Still getting "localhost" error?

CHECK 1: Is the variable actually added?
  → Go to Variables tab
  → You should SEE: DATABASE_URL = postgresql://...
  → If not, add it again

CHECK 2: Did Railway redeploy?
  → Go to Deployments tab
  → Check the timestamp of latest deployment
  → It should be AFTER you added the variable
  → If not, manually redeploy

CHECK 3: Clear browser cache
  → Press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  → Or open in Incognito/Private mode

CHECK 4: Check deployment logs
  → Go to Deployments tab
  → Click on latest deployment
  → Check logs for errors
