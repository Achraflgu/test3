================================================================================
METHOD 2: GIT CREDENTIAL MANAGER (AUTOMATIC BROWSER LOGIN) ⭐⭐⭐⭐
================================================================================

Why: Uses browser to sign in automatically
Time: 5 minutes
Difficulty: ★★☆☆☆ (Easy)

REQUIREMENT: Git for Windows (you probably have this)

STEPS:
  1. Open PowerShell in MSPORT folder
  2. Run: prepare-for-github-desktop.bat (prepares files)
  3. Run: git init
  4. Run: git add .
  5. Run: git commit -m "Initial commit"
  6. Run: git remote add origin https://github.com/Achraflgu/test3.git
  7. Run: git push -u origin main
  8. Browser opens - sign in to GitHub
  9. Done! ✅

Browser login - no token to copy!
Best for: Command line users

Integrated with your editor!

================================================================================
METHOD 4: SSH KEYS (MOST SECURE) ⭐⭐⭐
================================================================================

Why: Never enter password again
Time: 15 minutes (first time setup)
Difficulty: ★★★☆☆ (Medium)

CHECK IF YOU HAVE SSH KEYS:
  PowerShell: ls ~/.ssh

IF YOU HAVE id_rsa or id_ed25519:
  1. Copy public key: cat ~/.ssh/id_rsa.pub
  2. Add to GitHub: https://github.com/settings/keys
  3. Use: git@github.com:Achraflgu/test3.git

IF YOU DON'T:
  1. Create: ssh-keygen -t ed25519
  2. Add to GitHub: https://github.com/settings/keys
  3. Use SSH URL

Most secure - no passwords!
Best for: Advanced users, security-conscious
